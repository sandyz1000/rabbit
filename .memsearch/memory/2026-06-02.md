- User asked whether HTTP/2 streams require persistent connections with the client, after opening src/adapter/agent.rs in the IDE.
- Claude Code explained that HTTP/2 requires a persistent TCP connection and how multiple concurrent streams are multiplexed within that single connection using stream IDs.
- Claude Code clarified what "persistent" means in practice: TCP connection stays open while either side needs it, streams can open/close independently, and connection drops kill all streams simultaneously.
- Claude Code referenced the rabbit project's implementation (Channel::from_shared in adapter/agent.rs:52-54) where a single persistent TCP+HTTP/2 connection is established and each tunnel() call opens a new stream on that connection.
- Claude Code flagged a potential production concern: since tunnels in rabbit are long-lived, a single TCP connection drop is catastrophic and would require connection-level reconnect logic and stream-level re-registration (like ngrok/Cloudflare Tunnel use).
